## 2. 디벨롭안 — 5단계 로드맵

### D-1. 안정성 핫픽스 ★최우선 (버그 수정 — 소규모, 즉시 실행 권장)

데이터를 잃거나 게임이 멈추는 결함. 전부 국소 수정.

1. **세이브 파괴 방지**: migrate 실패 시 원본 문자열을 `potionworks-save-corrupt`로 보존 후에만 새 게임 시작 + "세이브를 읽지 못해 백업해 두었습니다" 안내. mana/basePower에도 `normalizeNonNeg`, generators에 정수·상한 검증(Infinity 연쇄 차단), serialize 직전 유한성 검사 (D3·D6·D8)
2. **에러 바운더리**: 최상위 1개 — 폴백 UI에 "세이브 내보내기" + "하드리셋" 버튼 (D8)
3. **hardReset 경합 수정**: 리셋 플로우에서 자동저장 리스너 정지 후 clearSave→reload (D1·D3)
4. **tick catch-up 캡**: `MAX_TICK_CATCHUP_MS`(제안 5분) 초과분은 offlineEarnings 공식으로 라우팅 — 오프라인 정책 우회·시계 조작 악용 차단 (D5). 음수 elapsed는 무시 대신 lastTick 재앵커(±5s 허용) — NTP 역행·simulate 정지 동시 해결 (D1·D5)
5. **60초 미만 부재 지급**: 팝업만 생략하고 마나는 100% 조용히 합산 (U3)
6. **모바일 헤더/터치**: ~480px 미디어쿼리로 헤더 2단 재배치, `touch-action: manipulation`+`user-select: none`+tap-highlight 제거 (U5)
7. **formulas 유한성 가드**: maxAffordable에 `!Number.isFinite(mana) → 0` + n 상한 클램프(무한루프 차단), stardustFor NaN 가드, 1개-구매 가드를 bulkCost 기준으로 통일 (D2)
8. **BurstEffect 누수**: 제거 타이머를 effect cleanup에서 분리(onAnimationEnd 방식) (D4)
9. **사운드 재생 try/catch** + `resume().catch()` (D8)

### D-2. UX 개선 (중규모 — 신뢰·정보·조작)

1. **공통 Modal 컴포넌트**: ESC 닫기·초기 포커스·포커스 트랩·닫힘 시 포커스 복귀·`aria-labelledby`·mousedown 기준 backdrop 판정. 5개 모달 이관 + 확인 UX 3종(`window.confirm` 포함)을 ConfirmModal로 통일 (U6·D7)
2. **정보 표시**: 시설 행에 배율 반영 실효 MPS·"총 X/s (전체의 N%)"·구매 시 "+Y/s" 델타. 솥 아래 "클릭당 +N" 상시 표시 (U8·U2)
3. **통계 패널**: 총 클릭/누적 마나(총·이번 생)/플레이 시간/각성 횟수/보너스 합산 내역(업적 +N%·스타더스트 +N%) (U8·U2)
4. **업적 진행도**: 잠긴 업적에 723/1,000 카운터, 목록 상단 보너스 합계, 조건 표기를 본편 포맷(1.00M/s)으로 통일 (U8)
5. **저장 신뢰 표시**: 자동저장 시각 갱신(또는 "자동 저장됨" 점멸), 저장 실패 시 1회성 경고 배너, 불러오기 성공 토스트, 하드리셋 1단계에 "내보내기 권장" 문구 (U4·D8)
6. **오프라인 팝업 정정**: 캡 적용 명시("10시간 부재 → 최대 8시간 정산"), 시간 표기 분·초 병기 (U3)
7. **온보딩 다듬기**: 각성 게이지는 누적 1만 도달 전 숨김+툴팁, 수량 토글은 시설 2종부터 노출, ×MAX "×0" → "×1 비활성" 표기, 비용에 마나 단위 표기 (U1·U2·U7)
8. **접근성 마감**: 시설 구매 버튼 aria-label(시설명+비용), 구매 후 포커스 유지(aria-disabled), 활성 토글·disabled 가격 대비 4.5:1 확보, Space 홀드 연타 (U6)
9. **CI 정비**: PR용 ci.yml(test+build), `--passWithNoTests` 제거, vitest 설정 명시, eslint+prettier(또는 biome) 도입, base 환경변수화 (D9)

### D-3. 각성 리워크 ★게임성 핵심 (설계 변경 — 승인 필요)

리뷰 최저점(U7 4점, U2 6점, U3 5.5점)이 모두 가리키는 지점: **스타더스트를 쓸 곳이 없다**.

1. **스타더스트 상점** (신규 시스템): 각성 화폐 소비처 —
   - 시작 부스트: "각성 후 견습생 N명 보유 시작" (리빌드 지루함 해소, U7)
   - 클릭 강화: "클릭 = MPS의 +1%p/레벨" (클릭 사망 해소, U10·U2)
   - **오프라인 효율 50→75%, 캡 8→12h** (방치 성장축 신설, U3·U2)
   - 전체 생산 % (기존 패시브를 상점 항목으로 흡수할지는 설계 시 결정)
2. **첫 각성 체감 보정**: 첫 각성 한정 최소 +3 보장(또는 스타더스트 곡선 재설계), 확인 모달에 "✨1→2 (+10%→+20%)" 전후 합계, **"다음 +1까지 4.00B" 상시 표시**, 임계 도달 후에도 누적 진행 계속 표시 (U7)
3. **각성 사전 설명**: 진행 게이지 툴팁/클릭 시 2줄 소개 (U7·U1)
4. **밸런스 조정(data)**: 마나 공명 II 500K→100K 이하 또는 효과 5%p로(가치 함정 제거), 5~15분 구간 소형 업그레이드 1~2개, 숙련 40단계 추가(50단계 앞 갭 해소), 40~55분 정체 완화 (U10)
5. 조정 후 T8.1 시뮬레이터로 목표표 재검증 (기존 방법론 재사용)

### D-4. 주스 패스 (연출 — U9 4.5점 해소)

1. **방치 앰비언트**: 버블 둥실 CSS 루프(2~4s), 솥 글로우 브리딩, MPS 비례 배경 기포 — "완전 정지 화면" 해소
2. **클릭 팝 개선**: 밝은 색+외곽선(대비), 연타 시 크기·각도 가중, N연타 솥 shake
3. **구매 피드백**: 행 플래시 0.3s + 보유수 팝, 마나 200~300ms 카운트 롤링
4. **각성 시퀀스**: 골드 플래시 → ✨가 헤더로 날아가 박힘 → 페이드인 (~1.5s)
5. **버스트 원점 고정**: 토스트 아이콘 또는 솥 중심으로, 토스트 위치·최대 스택 조정(모바일 구매 버튼 겹침·각성 직후 6개 스택 해소) (U5·U7)
6. **(승격 제안)** 골든 이벤트: 가끔 떠다니는 유성 클릭 → 30초 생산 ×7 — 주차장 항목이지만 U2·U10이 공통 지적한 "능동 플레이 이유 부재"의 최소 해법. D-3의 배율 컨텍스트(아래 D-5.2)가 선행되면 저비용

### D-5. 기술 부채·M9 대비 (승인 후, M9 착수 전)

1. **ID 리터럴 타입화**: `GeneratorId` 유니언 + 데이터 참조 무결성 단위 테스트, UpgradeEffect exhaustive check(never 가드) (D6)
2. **배율 컨텍스트 객체**: `recalcDerived` 위치 인자 5개 → 컨텍스트 1개, `composeGlobalMult` 순수 함수로 합성 일원화 — 골든 이벤트·매니저의 선행 조건 (D10)
3. **세이브 v4 메타**: `deviceId`·`playtimeMs`·`saveCount`(단조) — M9 충돌 해소를 클라이언트 시계에 의존하지 않게 (D10)
4. **저장 트리거 현대화**: `pagehide`+`visibilitychange(hidden)` 우선, beforeunload 보조, tick에 `pageshow` 복원 (D5)
5. **수식 property fuzz 테스트**: maxAffordable 라운드트립·단조성·분할 일관성, closed-form을 단일 진실로 명시 (D2)
6. **i18n 부채 동결**: 신규 코드부터 데이터 `nameKey` 관례 (D10)

## 3. 우선순위 근거

- **D-1이 최우선**: "몇 달 진행이 10초 만에 영구 소실"(D3 실증), "폰 첫 화면 파손"(U5)은 출시 상태에서 유저를 잃는 결함. 전부 국소 수정이라 리스크 낮음.
- **D-3이 게임성 핵심**: 최저점 3개 리뷰(각성 4, 정보 4, 방치 5.5)의 근본 원인이 동일 — 각성 화폐의 소비처 부재. 상점 하나가 세 문제를 동시에 푼다.
- D-2는 D-1과 병행 가능, D-4는 독립적, D-5는 M9 착수 직전이 적기.
